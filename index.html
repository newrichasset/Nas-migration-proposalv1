2. 전환 배경

2.1 환경 변화

합병 전

  • 소규모 팀 (3-4명)
  • 구글 자유 사용
  • 외부 클라우드 제약 없음

합병 후

  • 조직 확대 (5-7명)
  • 구글 사용 제한
  • 보안 심사 강화
  • 개인정보 정책 강화

2.2 현행 문제점

1. 외부 의존성

  • 구글 서비스 전적 의존
  • 정책 변경 시 작동 불가
  • 합병 회사 정책 충돌

2. 개인정보 보안

구글 시트 저장 정보:

  • 주민번호/생년월일
  • 연락처 (전화/이메일)
  • 주소
  • 사업자등록번호

리스크: 개인정보보호법 위반 가능성

3. 협업 불편

  • 수동 동기화 필요
  • 데이터 누락 가능성
  • 최신 데이터 확인 어려움
  • 동시 작업 충돌

4. 접근성 제한

  • PC에서만 작업 가능
  • Python 서버 필요
  • 배치파일 실행 필요
  • 담당자 PC 의존

2.3 전환 시급성

요인 시급성 설명
합병 통합 높음 3개월 내 구글 통합 예정
보안 정책 높음 즉시 개선 필요
업무 효율 중간 시간 손실

3. NAS 기반 시스템

3.1 전환 개요

구글 의존성
외부 서버, 수동 동기화
↓
NAS 전환
내부 서버, 자동 동기화
↓
보안 강화 + 효율 향상

변경 사항

항목 현행 개선
저장소 구글 시트 NAS
동기화 수동 자동
접속 로컬 서버 NAS 웹
백업 수동 자동
개인정보 외부 내부
협업 제한적 실시간

3.2 시스템 아키텍처

여러 사용자 (어디서나)
↓
NAS 웹 서버

HTML + API + 데이터
↓
자동 백업 + 버전 관리 
신규 데이터 흐름
작업자 A       NAS         직원 B/C/D
   │            │               │
   ├─ 접속      │               │
   ├─ 자동 로드 ← backup.json  │
   │            │               │
   ├─ 입력      │               │
   ├─ 자동 저장 → backup.json  │
   │       (날짜별 백업)         │
   │            │               │
   │            │ ←──── 접속 ───┤
   │       자동 로드 ───────────→│
   │            │       (최신)

3.3 핵심 기능

자동 동기화

  • 페이지 열면 자동 로드
  • 변경하면 자동 저장
  • 버튼 클릭 불필요
  • 항상 최신 유지

자동 백업

저장 시마다 자동 백업
/data/backup.json (최신)
/data/backup_2025-01-02_09-30-15.json
/data/backup_2025-01-02_14-15-32.json

- 날짜/시간별 백업
- 실수 복구 가능
- 히스토리 추적

접근성 향상

http://나스IP/groupbuy/

- PC 안 켜도 됨
- 배치파일 불필요
- Python 불필요
- NAS만 있으면 됨

보안 강화

  • NAS 내부 저장
  • 회사 네트워크 내
  • VPN 외부 접속 (선택)
  • 사용자 권한 관리
